@nicbat/svelte-masonry — interactive playground

A faithful vanilla-JS reimplementation of the component's algorithm: items join the currently shortest column, with height estimated from an aspect ratio (columnWidth / aspectRatio + footerEstimate) purely to pick a column — never written back as a fixed height. Resize the window or drag the controls.

Reading-order caveat: badges show that visual order (green) reads row-major, but DOM / Tab / screen-reader order (blue) runs column-major — down column 1, then column 2. Press Tab through the tiles to feel it. Fine for decorative galleries; a real concern for content whose order carries meaning. (See research doc §5.)
Wrong aspect ratios are active. Column balance gets worse, but notice every tile still renders its full content with no clipping or overlap — that's the whole point.
The tiles size themselves in normal flow; the script only decides which column each one lands in. That is exactly what src/lib/Masonry.svelte does.
